# @lionlang/contracts

Solidity implementation of the LionVM — a tree-walking interpreter that
executes Lion programs supplied in transaction calldata.

A caller encodes a Lion expression with `@lionlang/evm` and sends the
resulting bytes to `LionVM.execute(bytes)`. The contract decodes the TLV
wire format, evaluates the program against a fresh environment, and
returns the tagged result. Programs can read from and write to any other
contract via the `call` and `staticcall` builtins.

## Quick start

```bash
bun install                       # from repo root
cd packages/contracts
forge install foundry-rs/forge-std --no-commit   # once
bun run build
bun run test
```

## Wire format

Documented in `@lionlang/evm/tags`. Tag byte + payload:

| Tag  | Name   | Payload                                   |
| ---- | ------ | ----------------------------------------- |
| 0x01 | INT    | 32-byte two's-complement int256           |
| 0x02 | BOOL   | 1 byte (0/1)                              |
| 0x03 | SYM    | uint16 len + utf-8 (variable reference)   |
| 0x04 | BYTES  | uint16 len + raw bytes (string/bytes)     |
| 0x05 | LIST   | uint16 count + concatenated children      |
| 0x06 | NIL    | empty                                     |
| 0x07 | LAMBDA | runtime-only (never sent in calldata)     |
